### 詰まった事・注意事項
- ローカルHEADがリモートから分岐していることに気づかず `git push` しようとして `[rejected] (fetch first)`。前セッションの作業(Partner Center登録やstore:true化など)がこのセッションの会話コンテキストに引き継がれておらず、リモートの方が進んでいた。**push前は必ず `git fetch` + `git log HEAD..origin/main` で差分を確認する**
- 「mainをpushして」と言われて確認したところ、実際には差分ゼロで何もすることがなかった。未コミットのCLAUDE.md編集だけが残っていたが、後で「間違えた」と言われ破棄。ユーザーは別内容(日本語コミュニケーション指示を追加した版)で正式にコミットし直した。**pushを頼まれても機械的に実行せず、まず状態を確認してから動く**
- Hive初期化の不具合は、`path_provider` の `getApplicationSupportDirectory()` の戻り値を渡していても、Hive内部が独自に `getApplicationDocumentsDirectory()` を再取得する実装になっているのが原因。OneDriveなどでドキュメントフォルダがリダイレクトされ実体が無い環境特有の罠
- **releaseビルドは例外が起きても赤いエラー画面が出ず、灰色の空白ウィンドウになるだけ**(Flutterのデバッグ用エラー表示はdebugビルド専用機能で無効化されるため)。灰色ウィンドウ+エラーダイアログ無しを見たら、まずデバッグモードで再現させてスタックトレースを取るのが有効
- `git push`がGit Credential Managerの対話ログインを要求し、非対話シェル(自動化ツールのBash/PowerShell)からは永久にハングする。一度バックグラウンドプロセスを強制終了したところ、ユーザー側で進行中だったブラウザのOAuthコールバックが接続失敗になった。**GUI認証が絡む操作はユーザー自身のインタラクティブなターミナルでやってもらうしかない**
- MSIXの証明書信頼登録(`Cert:\LocalMachine\Root`への書き込み)はUAC同意が必要な操作で、管理者権限のPowerShellから`dart run msix:create`(`install_certificate: true`)を実行しても、内部の`Start-Process -Verb runAs`が対話UACを要求するため自動化できなかった。最終的に`New-SelfSignedCertificate`→`Export-PfxCertificate`→ユーザーが手動で`Import-PfxCertificate`という手順に落ち着いた
- 自動化ツールのシェルセッションはPATHやユーザー権限の変更を都度読み直さない(プロセスが使い回される)。`setx`やレジストリでPATHを追加しても同じセッション内では反映されず、以後`$env:Path += ";C:\dev\flutter\bin"`のように毎回明示的に補う必要があった
